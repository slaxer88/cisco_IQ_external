1. 소개 (Introduction)

Cisco IQ™는 자산 가시성 향상, 스마트한 인사이트 제공, 케이스 관리 간소화를 위한 기능을 고객에게 제공합니다. Cisco IQ AI Assistant와 같은 AI 기능은 운영 성과를 최적화하고, 사용자가 선제적이고 정보에 기반한 결정을 내릴 수 있도록 지원하며, 고객 참여 및 성공을 위한 프로세스를 간소화합니다.

Cisco IQ Link는 온프레미스 네트워크에서 자산 텔레메트리를 안전하게 수집하고 Cisco IQ로 전송하여, AI 기반 예측 인사이트를 통해 네트워크 가시성을 향상시키고, 장애를 사전에 예측하며, 운영 효율성을 높일 수 있도록 지원합니다.

2. Local Authentication (로컬 인증)

Administrator는 다음 자격 증명을 사용하여 Cisco IQ Link에 로그인합니다:

  • Default Username: admin
  • Default Password: Cisco IQ Link 설치 과정에서 설정한 비밀번호 (자세한 내용은 Cisco IQ Link Getting Started Guide 참조)

로그인 후 기본 사용자 "admin"과 계정명 "Default-Customer"가 홈 페이지에 표시됩니다.

2.1 Local Admin Security 설정

System Configuration의 Local Admin Security 메뉴에서 비밀번호를 변경하거나 보안 질문을 설정할 수 있습니다.

10분 이내에 비밀번호를 3번 잘못 입력하면 계정이 60분 동안 잠깁니다. 잠금 기간 중에는 로그인을 시도할 수 없으며, 시스템에 다음 메시지가 표시됩니다: "Account locked due to too many failed attempts. Please try again later." (잠금 해제 시간 포함). 60분 후 자동으로 잠금이 해제됩니다.

2.1.1 보안 질문 및 답변 설정 (Setting Up Security Questions and Answers)

보안 질문은 비밀번호를 분실했을 때 본인 확인에 사용됩니다. 비밀번호 재설정 기능을 활성화하려면 Administrator가 5개의 보안 질문에 대한 답변을 설정해야 합니다. 이는 최초 1회만 수행하면 됩니다.

보안 질문 설정 방법:

  1. System Settings에서 System Configuration > Local Admin Security > Security Questions 선택

    Security Questions

  2. Configure security questions 클릭

    Security Questions 구성
  3. 드롭다운 목록에서 5개의 보안 질문 선택
  4. 각 질문에 대한 답변 입력
  5. Save 클릭
Notes
• 답변은 대소문자를 구분하지 않습니다 (예: "SMITH"와 "smith"는 동일)
• 앞뒤 공백은 무시됩니다 (" Smith "와 "Smith"는 동일 처리)
• 답변을 업데이트할 경우, 이전 답변이 모두 교체되므로 변경하고 싶은 항목뿐만 아니라 5개 모두 다시 입력해야 합니다.

2.1.2 비밀번호 관리 (Managing Passwords)

Cisco IQ의 비밀번호는 로컬 Administrator만 관리할 수 있습니다.

사전 조건 (Prerequisites)
  • 로컬 Administrator 권한 보유
  • 로컬 Administrator 계정 사용 (SSO 또는 외부 인증 아님)
  • Cisco IQ에 로그인된 상태
  • 현재 비밀번호 알고 있음
비밀번호 변경 (Changing Passwords)

  1. System Settings에서 System Configuration > Local Admin Security > Change Password 이동

    Change Password
  2. 현재 Password 입력
  3. New password 입력
  4. 확인을 위해 새 비밀번호 재입력
  5. Save 클릭

변경된 비밀번호는 Cisco IQ Virtual Machine(VM)을 포함한 Cisco IQ 시스템 전체에 적용됩니다.

잊어버린 비밀번호 재설정 (Resetting a Forgotten Password)

보안 질문을 사전에 설정해 둔 경우, 보안 질문 인증 절차를 통해 비밀번호를 재설정할 수 있습니다.

  1. Cisco IQ Link 로그인 페이지로 이동

  2. Forgot Password 클릭

    Forgot Password
  3. Username 입력

  4. Continue 클릭 → Verify Identity 페이지에서 사전 설정된 5개 중 무작위 3개의 보안 질문이 표시됩니다.

    Verify Identity
    Note: 표시되는 보안 질문은 사용자마다 다릅니다.
  5. 표시된 3개 질문 모두 답변 입력

  6. Verify 클릭 → 답변이 일치하면 새 비밀번호 입력 화면으로 이동

    Reset Password
Notes
• 10분 이내에 보안 질문 답변을 3번 틀리면 계정이 60분 동안 잠깁니다.
• 잠금 기간 중 메시지: "Account locked due to too many failed verification attempts. Please try again later."
• 60분 후 자동 잠금 해제됩니다.
  1. New password 입력
  2. 비밀번호 재입력 (확인)
  3. Submit 클릭

3.2 ADFS IDP SAML SSO 구성

이 섹션에서는 Microsoft Active Directory Federation Services(ADFS)를 Cisco IQ의 SAML IDP로 구성하는 방법을 안내합니다.

사전 조건 (Prerequisites)

  • ADFS 6.0 이상 권장
  • Windows Server 2012 R2 이상
  • Active Directory 연동 구성 완료
  • ADFS에 SSL/TLS 인증서 설치
  • Cisco IQ Administrator 접근 권한
  • ADFS 서버(Windows Server)에 대한 관리자 접근 권한
  • ADFS 서버의 PowerShell 접근 권한
  • ADFS와 Cisco IQ 간의 네트워크 연결
  • ADFS 서버 구성 정보 (아래 표 참조)

ADFS 서버 구성 (Configuring ADFS Servers)

  1. System Settings에서 System Configuration > Identity Providers 선택

  3. service-provider-metadata.xml 및 service-provider-certificate.crt 파일을 ADFS 디렉터리에 저장 (예: C:\)
  4. ADFS 서버에 로그인
  5. ADFS Management 메뉴에서 Relying Party Trusts 클릭
  6. Add Relying Party Trusts 클릭 → 새 마법사 열림
  7. Claims Aware 라디오 버튼 선택
  8. Start 클릭하여 구성 진행

  10. Browse를 클릭하여 서비스 공급자 메타데이터 파일 선택 및 업로드
  11. Next 클릭

  13. Choose Access Control Policy 페이지에서 Permit everyone 클릭 (또는 조직 보안 정책 선택)
  14. 나머지 화면에서 Next 클릭
  15. Close를 클릭하여 Relying Party Trust 구성 완료

Claim Rules 적용 (Applying Claim Rules)

1. Relying Party Trust 이름 정의:

$relyingPartyName = "Cisco IQ - Stage"

2. 사용자 정보 및 그룹 멤버십을 Cisco IQ로 보내기 위한 Claim Rules 정의:

$claimRules = @'
@RuleTemplate = "LdapClaims"
@RuleName = "Send Email and Name"
c:[Type == "http://schemas.microsoft.com/ws/2008/06/identity/claims/windowsaccountname", Issuer == "AD AUTHORITY"]
=> issue(store = "Active Directory", types = ("http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress", "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name"), query = ";mail,displayName;{0}", param = c.Value);

@RuleName = "Transform Email to NameID"
c:[Type == "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"]
=> issue(Type = "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/nameidentifier", Issuer = c.Issuer, OriginalIssuer = c.OriginalIssuer, Value = c.Value, ValueType = c.ValueType, Properties["http://schemas.xmlsoap.org/ws/2005/05/identity/claimproperties/format"] = "urn:oasis:names:tc:SAML:1.1:nameid-format:emailAddress");

@RuleName = "Send Group Membership"
c:[Type == "http://schemas.microsoft.com/ws/2008/06/identity/claims/windowsaccountname", Issuer == "AD AUTHORITY"]
=> issue(store = "Active Directory", types = ("http://schemas.xmlsoap.org/claims/Group"), query = ";memberOf;{0}", param = c.Value);
'@

3. Claim Rules 적용:

Set-AdfsRelyingPartyTrust -TargetName $relyingPartyName -IssuanceTransformRules $claimRules
사용자 그룹 확인 (Verifying User Groups)

1. 확인할 사용자 이름 설정:

$username = "testuser"

2. 사용자 계정 찾기:

$searcher = [adsisearcher]"(samaccountname=$username)"
$user = $searcher.FindOne()

3. 사용자가 속한 그룹 표시:

$user.Properties.memberof

출력 예시:

CN=Role - CXIQ Developers,OU=Role Groups,DC=dev,DC=local

ADFS가 SP Signing Certificate를 신뢰하도록 구성

1. ADFS 서버에서 SP 인증서를 TrustedPeople 저장소로 가져오기:

Import-Certificate -FilePath "C:\service-provider-certificate.crt" -CertStoreLocation "Cert:\LocalMachine\TrustedPeople"

2. 다음 중 하나를 선택하여 실행:

Note: SP 인증서는 ADFS가 표준 신뢰 체인으로 검증할 수 없는 내부 Certificate Authority에서 발급됩니다.

옵션 A - 이 Relying Party에 대한 체인 검증 전역 비활성화:

Set-AdfsRelyingPartyTrust `
    -TargetIdentifier "<sp_entity_id>" `
    -SigningCertificateRevocationCheck None `
    -EncryptionCertificateRevocationCheck None

옵션 B - 발급 CA 인증서를 Trusted Root Certification Authorities 저장소로 가져오기:

Import-Certificate -FilePath "C:\cisco-iq-onprem-ca.cer" -CertStoreLocation "Cert:\LocalMachine\Root"

3. ADFS 서비스 재시작:

Restart-Service adfssrv

ADFS Metadata 내보내기 (Exporting ADFS Metadata)

PowerShell 사용
  1. ADFS 서버에서 PowerShell 열기
  2. 다음 명령으로 메타데이터 파일 다운로드:
$metadataUrl = (Get-AdfsEndpoint | Where-Object {$_.Protocol -eq "Federation Metadata"}).FullUrl
Invoke-WebRequest -Uri $metadataUrl.AbsoluteUri -OutFile "C:\adfs-metadata.xml"
Write-Host "ADFS metadata exported to C:\adfs-metadata.xml" -ForegroundColor Green
웹 브라우저 사용
  1. https://<your-adfs-server>/FederationMetadata/2007-06/FederationMetadata.xml로 이동
  2. <your-adfs-server>를 실제 ADFS 서버 호스트명으로 교체
  3. 프롬프트 시 메타데이터 XML 파일을 컴퓨터에 저장

ADFS IDP 추가 (Adding ADFS IDP)

  1. Identity Providers 페이지에서 Add identity provider 클릭
  2. Identity provider 이름 입력
  3. Domain 입력 (예: company.com)
  4. (선택 사항) 필요한 경우 Enable single logout toggle 버튼 켜기
  5. Upload IDP Metadata field에 SAML metadata 파일 드래그 앤 드롭 또는 업로드
  6. Save 클릭

Role Mapping 구성 (Configuring Role Mapping)

Role Mapping 구성 전에 Active Directory에서 매핑에 사용할 그룹을 찾습니다:

$searcher = New-Object DirectoryServices.DirectorySearcher
$searcher.Filter = "(&(objectClass=group)(cn=Role - CXIQ*))"
$searcher.PropertiesToLoad.Add("distinguishedName") | Out-Null
$searcher.PropertiesToLoad.Add("cn") | Out-Null
$searcher.FindAll() | ForEach-Object { $_.Properties["distinguishedname"] }

그룹 정보는 전체 Distinguished Name(DN) 형식으로 반환됩니다. 예:

CN=Role - CXIQ Developers,OU=Groups,DC=dev,DC=example,DC=com
CN=Role - CXIQ Viewers,OU=Groups,DC=dev,DC=example,DC=com
Note: 필요한 그룹이 목록에 없으면 Administrator가 Active Directory에서 그룹을 먼저 생성해야 합니다.

  1. 추가된 IDP에서 More Options 아이콘 > Map Roles 선택

    Role Mapping
  2. 선택한 System role에 IDP role 입력:
    • general_account_administrator: 전체 권한. IDP Role (파싱된 이름): CXIQ Admins
    • general_account_viewer: 읽기 전용 접근. IDP Role (파싱된 이름): CXIQ Developers, CXIQ Viewers
    Note: 전체 Domain Name이 아닌 파싱된 이름(예: CXIQ Developers) 사용
  3. Save 클릭 → 상태가 Success로 업데이트됨

10. Connection Management (데이터 수집)

Cisco IQ Link는 네트워크 데이터 수집을 위한 온프레미스 배포 솔루션으로, 인프라에 대한 깊은 가시성을 제공합니다. Catalyst Center와 Direct Connection을 통해 데이터를 수집합니다.

데이터 수집 구성 요약:

  • Credential Sets 생성: 네트워크 장치와 통신하기 위한 인증 프로토콜(예: SNMP v1/v2c/v3) 설정. 보안 구역 또는 위치별로 자격 증명을 중앙화하면 한 곳에서 비밀번호를 업데이트할 때 연결된 모든 장치에 자동으로 적용됩니다.
  • Inventory에 Credential 매핑: Credential Sets를 Inventory Assets에 매핑하여 인증 프로세스를 자동화합니다. 특정 IP 범위를 정의된 Credential Sets에 연결하는 규칙을 생성하면 시스템이 데이터 수집 중 올바른 인증을 자동으로 적용합니다.
Note: 장치 검색에는 SNMPv2c/SNMPv3 및 SSH가 필요하며, Catalyst Center 구성 전에 HTTP/HTTPS 자격 증명을 먼저 입력해야 합니다.

Credential 선택 및 매칭 로직 (Credential Selection and Matching Logic)

텔레메트리 엔진은 검색 및 수집 시 우선순위 기반 매칭 로직을 사용하여 적용할 자격 증명을 결정합니다.

  • 우선순위: Exact IP match (최높 우선순위) > Trailing Wildcard Match (별표가 적을수록 더 구체적)

    • 지원: 1.2.3.* / 1.2.*.* / 1.*.*.* / *.*.*.*
    • 미지원: 앞쪽 와일드카드, 옥텟 사이 와일드카드, 대시 등

  7. Banner location 선택
  8. Save 클릭 → 배너가 애플리케이션 전체에 표시됩니다.

12. Troubleshooting (문제 해결)

고객은 Cisco IQ 시스템에서 진단 및 로그 파일을 수집하여 SCP 서버로 안전하게 전송할 수 있습니다. 이 파일들은 문제를 보고할 때 지원팀과 공유하여 문제 해결에 도움을 줄 수 있습니다.

  1. Cisco IQ에 로그인

    Main Menu

  2. Cisco IQ Main Menu에서 "3"을 입력하고 Enter를 눌러 System Diagnostics 선택

    System Diagnostics
  3. SCP/SFTP Server Address 입력
  4. SCP/SFTP Server Port 입력
  5. SCP/SFTP Server Path 입력
  6. 프로토콜 선택
  7. Username 입력
  8. Password 입력

  9. "C"를 입력하고 Enter를 눌러 시스템 진단 계속

    System Diagnostic Operation Complete

시스템이 진단 프로세스를 시작하고 다음 작업을 수행합니다:

  • Reachability 확인
  • 시스템 정보 수집
  • Kubernetes 정보 수집
  • 로그 수집
  • System Diagnostics 번들 준비
  • System Diagnostics 번들 업로드

완료 후 생성된 번들 이름이 포함된 확인 메시지가 표시됩니다.

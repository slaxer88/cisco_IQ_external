Cisco IQ Link Installation Guide

Official installation guide for customer operations teams (English Edition)  |  Last updated: April 2026

🌐

What is Cisco IQ?

An AI-powered unified network management platform that provides asset visibility, vulnerability detection, and Support integration through a single interface.

🔗

What is Cisco IQ Link?

An On-Premises Connector deployed in the customer's Data Center. It securely collects Telemetry data from network devices and sends it to Cisco IQ SaaS.

🤖

AI Assistant

Provides predictive insights based on operational data to help detect issues proactively and improve operational efficiency.

Architecture Overview

Cisco IQ Link resides within the customer's internal network, collects data from network devices, and securely transmits it to the Cisco IQ Cloud over HTTPS (443).

Customer Network
Devices (NW Devices)
→
Cisco IQ Link
(On-Premises VM)
→ HTTPS 443 →
Cisco IQ
SaaS Cloud
→
Operations Team
(Web Browser)
ℹ️ IQ Link uses one-way (outbound-only) communication. There is no inbound access from outside into the customer's internal network.

Supported Hypervisors

  • VMware ESXi v8.0 or later + VMware vSphere Web Client v8.0 or later
  • Hyper-V on Microsoft Server 2022 / 2025
  • KVM (based on RHEL v9.7)
⚠️
Thick Provisioning recommended: Using Thin Provisioning can cause service disruption due to storage overcommitment. Disk write speed must be at least 70 MB/s.

Prerequisites — Network Requirements

IP / DNS Settings

  • A fixed IPv4 address is required (DHCP is not supported)
  • DNS A record: hostname → IPv4 address mapping required (example: ciscoiqlink.mydomain.com)
  • DNS PTR record: reverse lookup support required

Cisco IQ SaaS Connectivity FQDNs

Outbound connectivity must be allowed to the FQDNs that match the region where IQ Link is installed.

VM Installation

Refer to the appropriate procedure in the tabs below, based on the Hypervisor used in your environment.

ℹ️ IQ Link is provided as an OVA file. It must be deployed through VMware vCenter; direct deployment to ESXi is not supported.
  1. Log in to VMware vSphere Web Client with an administrator account.
  2. Right-click the appropriate vCenter object (Data Center, cluster, or ESXi host) and select Deploy OVF Template.
  3. Specify the OVA file path (URL or local path) and click Next.
  4. Review the OVF Template information and click Next.
  5. Specify the VM name and location, then click Next.
  6. Select the ESXi host, cluster, or Resource Pool for deployment, then click Next.
  7. Review the OVA details and click Next.
  8. Select the deployment configuration and click Next.
  9. Select the storage location for the VM files. Disk Format should use Thick Provisioning when possible.
  10. Map the network and click Next.
  11. On the Ready to Complete screen, check Power On After Deployment and click Finish.
  12. Right-click the VM → open Edit Settings to apply additional settings.
    • CPU Shares: select Low
    • Memory: check Reserve all guest memory (All locked)
    • Set CPU/RAM based on the hardware requirements table
  13. Select the VM → Summary tab → click Launch Console to open the console.
  14. Go to the Network Configuration section to continue.
ℹ️ The Hyper-V package is provided as a .tar.gz file. After extraction, place the virtual hard disk files in the designated folder.
  1. Log in to Hyper-V Server Manager with an administrator account.
  2. Extract the IQ Link .tar.gz package into the Virtual Hard Disks folder.
  3. Verify that all disk files are present in the folder.
  4. In Hyper-V Manager → Actions panel → select New > Virtual Machine and click Next.
  5. Enter the VM name and click Next.
  6. For generation, select Generation 2.
  7. Enter the recommended memory value. Leave Use Dynamic Memory unchecked.
  8. Select the appropriate network adapter.
  9. Attach the first virtual hard disk (Disk 1).
  10. Review the summary and click Finish.
  11. Right-click the VM → open Settings and apply the following.
    • Security → clear Enable Secure Boot
    • Add the remaining two disks (repeat Add Hardware)
    • Network Adapter → Advanced Features → check Enable device naming
    • Processors → set to the recommended vCPU count
  12. In the Actions panel, click Start to power on the VM.
  13. In the Actions panel, click Connect to access the VM console.
  14. Go to the Network Configuration section to continue.
ℹ️ KVM installation uses Virtual Machine Manager (VMM) on a host based on RHEL v9.7.
  1. Log in to the RHEL host server with an administrator account.
  2. Download the IQ Link package to the host and extract it.
  3. Launch the Virtual Machine Manager (VMM) client.
  4. In VMM, select File > New Virtual Machine.
  5. Select Import existing disk image and click Forward.
  6. Click Browse and create a Storage Pool using the extracted IQ Link package path.
  7. Select the first disk and click Choose Volume.
  8. Select AlmaLinux 9 as the OS type.
  9. Enter memory and CPU values according to the hardware requirements.
  10. Enter the VM name and select Customize configuration before install. Also select the virtual network.
  11. Click Finish to complete adding the first disk.
  12. Use Add Hardware → Storage to add the remaining two disks one by one.
  13. Set the Bus Type of all disks to SCSI.
  14. In the Overview section, set Firmware to UEFI.
  15. Click Begin installation.
  16. Complete the password and IP configuration in the VMM console.
  17. Go to the Network Configuration section to continue.

Network Configuration (VM Console)

After the VM boots, configure the network from the console using the steps below.

  1. Access the VM console. When the login screen appears, enter admin for both the username and password.
    ⚠️ These are the default credentials for initial setup only. Change the password immediately after installation is complete.
  2. When the Main Menu appears, enter "1" and press Enter to open network configuration.
  3. Enter the following network information in order. If a detected value is shown, you can press Enter to keep it.
    ItemDescription
    IP addressFixed IPv4 address to assign to the IQ Link VM
    Gateway IPDefault gateway address
    DNS IP listDNS server addresses (multiple entries supported)
    Search domainDNS Search Domain (example: mydomain.com)
    NTP server listNTP server addresses (multiple entries allowed, separated by commas)
  4. Select the NTP authentication setting.
    • N: continue without authentication
    • Y → 1: traditional key-based authentication (requires algorithm + hexadecimal key input)
    • Y → 2: use the NTS (Network Time Security) protocol
  5. Review the password requirements shown on screen, then set and confirm the new password.
  6. Review the configuration summary and enter Y to apply it. This may take a few minutes.
  7. After confirming the success message, press Enter to return to the Main Menu.
✅
Recommendation: After completing network configuration, create a Snapshot of the current VM state in vCenter (or the relevant Hypervisor). You can also continue additional work by connecting over SSH to the newly configured IP address.

Cisco IQ Link Installation (Web Installer)

ℹ️
Prerequisite: Before starting installation, first begin adding the IQ Link instance in the Cisco IQ portal.
Path: Cisco IQ → Home → System Settings → Data Connectors → Add Cisco IQ Link
  1. After completing VM network configuration, open the IQ Link Installer page in a web browser.
    (Use the URL provided during the VM configuration process.)
  2. Select the region from the Region dropdown.
    ℹ️ Customers in Korea should select APJC.
  3. Click the Start button. The Check Internet Connectivity tab appears.
  4. If a Proxy is required, enter the information below. If your environment uses direct connectivity, skip this step.
    • Proxy Hostname / IP
    • Port number
    • If authentication is required, enable the Enable Authentication toggle and enter Username/Password
  5. Click the Test Connectivity button to validate internet connectivity.
  6. Click Next. The Connect Cisco IQ Link tab displays the Device ID and Claim code.
    ⚠️ The Device ID and Claim code are automatically refreshed after 5 minutes. Proceed quickly to the next step.
  7. In the browser, go to the Cisco IQ portal. Confirm that the Add Cisco IQ Link page is open.
  8. Select the "I am ready to pair with an installed Cisco IQ Link" checkbox and click Next.
  9. Enter a unique name to identify this IQ Link instance and click Next.
  10. Enter the Device ID and Claim code shown in the IQ Link Installer, then click Submit. You will be redirected to the Data Connectors page.
  11. Return to the IQ Link Installer web page. The Check Network Requirements page appears.
  12. Click Next. The Configure Internal Network tab appears.
  13. Enter the Internal Network IP range.
    ⚠️ This IP range is reserved for IQ Link internal communication. Make sure it does not overlap with the IPs of external devices (DNS, NTP, data sources, and so on). If it overlaps, those devices will not be reachable.
  14. Click Next. The Installation Result page appears, and installation is complete.
  15. After installation completes, the Cisco IQ Link login page opens in the browser. Log in using the admin account and password configured during network setup.
    ✅ If the home screen appears after login, the installation has completed successfully.
ℹ️ Afterward, add network device connections in IQ Link under System Settings → Connection Management.

Installation Checklist

Pre-installation Checks

  • Determine VM specifications (vCPU/RAM/Storage based on device count)
  • Secure a fixed IP address and register DNS A/PTR records
  • Allow outbound ports 443 and 22 on the firewall (including UDP 53, 123, and 161)
  • Whitelist APJC FQDNs: ap-southeast-2.iq.cisco.com, ng.acs.agent.apjc.csco.cloud
  • Confirm Proxy information (IP, Port, whether authentication is required)
  • Complete download of the IQ Link OVA / .tar.gz / KVM package
  • Confirm Cisco IQ portal access permissions
  • Allocate 600 GB SSD Thick Provisioning storage
  • Confirm disk write speed is at least 70 MB/s

Post-installation Checks

  • Successful login to the IQ Link Web UI
  • New IQ Link instance appears in the Data Connectors list in the Cisco IQ portal
  • System Settings → NTP synchronization status is normal
  • VM Snapshot created successfully
  • Initial admin password changed

Troubleshooting

SymptomCauseAction
Internet connectivity test fails Firewall blocking, Proxy configuration error Check firewall policy, re-enter Proxy IP/Port/authentication information
Device ID / Claim code expired Automatically refreshed after 5 minutes Check the new code on the IQ Link Installer page and re-enter it
DNS lookup fails DNS server not configured or missing A/PTR records Recheck the DNS IP, create the A record and PTR record
NTP synchronization fails UDP 123 blocked or NTP server not responding Verify NTP server reachability and allow port 123
Storage capacity warning Overcommitment caused by Thin Provisioning Redeploy using Thick Provisioning
Internal Network IP conflict The entered IP range overlaps with existing devices Check the IP ranges already in use and reconfigure with a non-overlapping range
ℹ️ If additional assistance is required, open a Cisco Support Case. A Cisco Service Contract is required.

한국 고객 운영팀을 위한 공식 설치 가이드 (Korean Edition)  |  최종 업데이트: 2026년 4월

🌐

Cisco IQ란?

AI 기반의 통합 네트워크 관리 플랫폼으로, 자산 가시성 향상, 취약점 탐지, Support 연동을 하나의 인터페이스에서 제공합니다.

🔗

Cisco IQ Link란?

고객 Data Center 내에 설치되는 On-Premises Connector입니다. 네트워크 장비의 Telemetry 데이터를 안전하게 수집하여 Cisco IQ SaaS로 전송합니다.

🤖

AI Assistant

운영 데이터를 기반으로 예측 인사이트를 제공하고, 장애 사전 감지 및 운영 효율화를 지원합니다.

아키텍처 개요

Cisco IQ Link는 고객 내부망에 위치하며, 네트워크 장비로부터 데이터를 수집한 후 HTTPS(443)를 통해 Cisco IQ Cloud로 안전하게 전송합니다.

ℹ️ IQ Link는 단방향(아웃바운드) 통신만 사용합니다. 외부에서 고객 내부망으로의 인바운드 접속은 없습니다.

사전 준비 — 하드웨어 / VM 리소스

관리 대상 장비(Device) 수량에 따라 아래 표를 참고하여 VM 사양을 결정하십시오.

리소스 10K Devices 20K Devices 50K Devices
vCPU161632
RAM28 GB28 GB48 GB
Storage (SSD 권장)600 GB600 GB600 GB
수집 주기24시간 이상24시간 이상7일 이상

지원 Hypervisor

  • VMware ESXi v8.0 이상 + VMware vSphere Web Client v8.0 이상
  • Hyper-V on Microsoft Server 2022 / 2025
  • KVM (RHEL v9.7 기반)
⚠️
Thick Provisioning 권장: Thin Provisioning 사용 시 스토리지 과할당으로 인해 서비스 장애가 발생할 수 있습니다. 디스크 쓰기 속도는 반드시 70 MB/s 이상이어야 합니다.

사전 준비 — 네트워크 요구사항

IP / DNS 설정

  • 고정 IPv4 주소 필수 (DHCP 미지원)
  • DNS A 레코드: 호스트명 → IPv4 주소 매핑 필요 (예: ciscoiqlink.mydomain.com)
  • DNS PTR 레코드: 역방향 조회(Reverse Lookup) 지원 필요

VM 설치

고객 환경의 Hypervisor 종류에 따라 아래 탭에서 해당 절차를 참고하십시오.

ℹ️ IQ Link는 OVA 파일 형태로 제공됩니다. 반드시 VMware vCenter를 통해 배포해야 하며, ESXi에 직접 배포하는 방식은 지원되지 않습니다.
  1. VMware vSphere Web Client에 관리자 계정으로 로그인합니다.
  2. 적절한 vCenter 오브젝트(데이터센터, 클러스터, ESXi 호스트)를 우클릭하고 Deploy OVF Template을 선택합니다.
  3. OVA 파일의 경로(URL 또는 로컬 경로)를 지정하고 Next를 클릭합니다.
  4. OVF Template 정보를 확인하고 Next를 클릭합니다.
  5. VM 이름과 위치(Location)를 지정하고 Next를 클릭합니다.
  6. 배포할 ESXi 호스트, 클러스터 또는 Resource Pool을 선택하고 Next를 클릭합니다.
  7. OVA 세부 정보를 검토하고 Next를 클릭합니다.
  8. 배포 구성(Configuration)을 선택하고 Next를 클릭합니다.
  9. VM 파일을 저장할 스토리지를 선택합니다. Disk Format은 Thick Provisioning을 권장합니다.
  10. 네트워크를 매핑하고 Next를 클릭합니다.
  11. Ready to Complete 화면에서 Power On After Deployment를 체크하고 Finish를 클릭합니다.
  12. VM을 우클릭 → Edit Settings에서 추가 설정을 진행합니다.
    • CPU Shares: Low 선택
    • Memory: Reserve all guest memory (All locked) 체크
    • CPU/RAM은 하드웨어 요구사항 표를 참고하여 설정
  13. VM 선택 → Summary 탭 → Launch Console을 클릭하여 콘솔을 시작합니다.
  14. 네트워크 설정 섹션으로 이동하여 계속 진행합니다.
ℹ️ Hyper-V 패키지는 .tar.gz 파일로 제공됩니다. 압축 해제 후 가상 하드 디스크 파일을 지정된 폴더에 위치시킵니다.
  1. Hyper-V Server Manager에 관리자 계정으로 로그인합니다.
  2. IQ Link .tar.gz 패키지를 Virtual Hard Disks 폴더에 압축 해제합니다.
  3. 모든 디스크 파일이 폴더에 있는지 확인합니다.
  4. Hyper-V Manager → Actions 패널 → New > Virtual Machine을 선택하고 Next를 클릭합니다.
  5. VM 이름을 입력하고 Next를 클릭합니다.
  6. 세대는 Generation 2를 선택합니다.
  7. 권장 메모리 값을 입력합니다. Use Dynamic Memory는 체크 해제 상태로 유지합니다.
  8. 적절한 네트워크 어댑터를 선택합니다.
  9. 첫 번째 가상 하드 디스크(Disk 1)를 연결합니다.
  10. Summary를 확인하고 Finish를 클릭합니다.
  11. VM 우클릭 → Settings 진입 후 아래 설정을 적용합니다.
    • Security → Enable Secure Boot 체크 해제
    • 나머지 디스크 2개 추가 (Add Hardware 반복)
    • Network Adapter → Advanced Features → Enable device naming 체크
    • Processors → 권장 vCPU 수로 설정
  12. Actions 패널 → Start로 VM을 시작합니다.
  13. Actions 패널 → Connect로 VM 콘솔에 접속합니다.
  14. 네트워크 설정 섹션으로 이동하여 계속 진행합니다.
ℹ️ KVM 설치는 RHEL v9.7 기반 호스트에서 Virtual Machine Manager(VMM)를 사용합니다.
  1. RHEL 호스트 서버에 관리자 계정으로 로그인합니다.
  2. IQ Link 패키지를 호스트에 다운로드하고 압축을 해제합니다.
  3. Virtual Machine Manager(VMM) 클라이언트를 실행합니다.
  4. VMM에서 File > New Virtual Machine을 선택합니다.
  5. Import existing disk image를 선택하고 Forward를 클릭합니다.
  6. Browse를 클릭하여 IQ Link 패키지 압축 해제 경로로 Storage Pool을 생성합니다.
  7. 첫 번째 디스크를 선택하고 Choose Volume을 클릭합니다.
  8. OS 유형으로 AlmaLinux 9를 선택합니다.
  9. 메모리와 CPU를 하드웨어 요구사항에 맞게 입력합니다.
  10. VM 이름을 입력하고 Customize configuration before install을 선택합니다. 가상 네트워크도 선택합니다.
  11. Finish를 클릭하여 첫 번째 디스크 추가를 완료합니다.
  12. Add Hardware → Storage로 나머지 디스크 2개를 순차적으로 추가합니다.
  13. 모든 디스크의 Bus Type을 SCSI로 설정합니다.
  14. Overview 섹션에서 Firmware를 UEFI로 설정합니다.
  15. Begin installation을 클릭합니다.
  16. VMM 콘솔에서 패스워드 및 IP 설정을 완료합니다.
  17. 네트워크 설정 섹션으로 이동하여 계속 진행합니다.

네트워크 설정 (VM 콘솔)

VM이 부팅된 후 콘솔에서 아래 절차에 따라 네트워크를 구성합니다.

  1. VM 콘솔에 접속합니다. 로그인 화면이 표시되면 아이디/패스워드를 모두 admin으로 입력합니다.
    ⚠️ 초기 설정 전용 기본 자격증명입니다. 설치 완료 후 반드시 비밀번호를 변경하십시오.
  2. Main Menu가 표시되면 "1"을 입력하고 Enter를 눌러 네트워크 설정으로 진입합니다.
  3. 아래 네트워크 정보를 순서대로 입력합니다. 감지된 값이 있으면 Enter로 그대로 사용할 수 있습니다.
    항목설명
    IP addressIQ Link VM에 할당할 고정 IPv4 주소
    Gateway IP기본 게이트웨이 주소
    DNS IP listDNS 서버 주소 (복수 입력 가능)
    Search domainDNS Search Domain (예: mydomain.com)
    NTP server listNTP 서버 주소 (쉼표로 구분하여 복수 입력 가능)
  4. NTP 인증 설정을 선택합니다.
    • N: 인증 없이 진행
    • Y → 1: 전통적 키 기반 인증 (알고리즘 + 16진수 키 입력 필요)
    • Y → 2: NTS (Network Time Security) 프로토콜 사용
  5. 화면의 패스워드 조건을 확인한 후 새 비밀번호를 설정하고 확인 입력을 합니다.
  6. 설정 요약을 검토하고 Y를 입력하여 적용합니다. 몇 분 정도 소요될 수 있습니다.
  7. 성공 메시지 확인 후 Enter를 눌러 Main Menu로 돌아옵니다.
✅
권장 사항: 네트워크 설정 완료 후 vCenter(또는 해당 하이퍼바이저)에서 현재 VM 상태의 Snapshot을 생성해 두십시오. 이후 SSH를 통해 새로 설정한 IP로 접속하여 추가 작업을 진행할 수도 있습니다.

Cisco IQ Link 설치 (Web Installer)

ℹ️
사전 조건: 설치 시작 전 Cisco IQ 포털에서 IQ Link 인스턴스 추가를 먼저 시작해야 합니다.
경로: Cisco IQ → Home → System Settings → Data Connectors → Add Cisco IQ Link
  1. VM 네트워크 설정 완료 후 웹 브라우저에서 IQ Link Installer 페이지에 접속합니다.
    (VM 구성 과정에서 안내된 URL 사용)
  2. Region 드롭다운에서 리전을 선택합니다.
    ℹ️ 한국 고객은 APJC를 선택합니다.
  3. Start 버튼을 클릭합니다. Check Internet Connectivity 탭이 표시됩니다.
  4. Proxy가 필요한 경우 아래 정보를 입력합니다. 직접 연결 환경이라면 이 단계를 건너뜁니다.
    • Proxy Hostname / IP
    • Port 번호
    • 인증이 필요한 경우 Enable Authentication 토글 활성화 후 Username/Password 입력
  5. Test Connectivity 버튼을 클릭하여 인터넷 연결을 검증합니다.
  6. Next를 클릭하면 Connect Cisco IQ Link 탭에 Device ID와 Claim code가 표시됩니다.
    ⚠️ Device ID와 Claim code는 5분 후 자동 갱신됩니다. 다음 단계를 신속하게 진행하십시오.
  7. 브라우저에서 Cisco IQ 포털로 이동합니다. Add Cisco IQ Link 페이지가 열려 있는지 확인합니다.
  8. "I am ready to pair with an installed Cisco IQ Link" 체크박스를 선택하고 Next를 클릭합니다.
  9. 이 IQ Link 인스턴스를 식별할 고유한 이름을 입력하고 Next를 클릭합니다.
  10. IQ Link Installer에서 확인한 Device ID와 Claim code를 입력하고 Submit을 클릭합니다. Data Connectors 페이지로 이동됩니다.
  11. IQ Link Installer 웹 페이지로 돌아옵니다. Check Network Requirements 페이지가 표시됩니다.
  12. Next를 클릭합니다. Configure Internal Network 탭이 표시됩니다.
  13. Internal Network IP 범위를 입력합니다.
    ⚠️ 이 IP 범위는 IQ Link 내부 통신에 예약됩니다. 외부 장비(DNS, NTP, 데이터 소스 등)의 IP와 중복되지 않도록 주의하십시오. 중복 시 해당 장비에 접근할 수 없습니다.
  14. Next를 클릭합니다. Installation Result 페이지가 표시되며 설치가 완료됩니다.
  15. 설치 완료 후 브라우저에 Cisco IQ Link 로그인 페이지가 열립니다. 네트워크 설정 시 지정한 admin 계정과 패스워드로 로그인합니다.
    ✅ 로그인 후 홈 화면이 표시되면 설치가 성공적으로 완료된 것입니다.
ℹ️ 이후 네트워크 장비 연결(Connection) 추가는 IQ Link 내 System Settings → Connection Management에서 진행합니다.

설치 체크리스트

설치 전 확인사항

  • VM 사양 결정 (Device 수량에 따른 vCPU/RAM/Storage)
  • 고정 IP 주소 확보 및 DNS A/PTR 레코드 등록
  • 방화벽에서 아웃바운드 포트 443, 22 허용 (UDP 53, 123, 161 포함)
  • APJC FQDN 화이트리스트 등록: ap-southeast-2.iq.cisco.com, ng.acs.agent.apjc.csco.cloud
  • Proxy 정보 확인 (IP, Port, 인증 여부)
  • IQ Link OVA / .tar.gz / KVM 패키지 다운로드 완료
  • Cisco IQ 포털 접속 권한 확인
  • SSD Thick Provisioning 600GB 스토리지 할당
  • 디스크 쓰기 속도 70 MB/s 이상 확인

설치 완료 확인사항

  • IQ Link Web UI 로그인 성공
  • Cisco IQ 포털의 Data Connectors 목록에 신규 IQ Link 인스턴스 표시
  • System Settings → NTP 동기화 상태 정상 확인
  • VM Snapshot 생성 완료
  • admin 초기 비밀번호 변경 완료

문제 해결

증상원인조치
인터넷 연결 테스트 실패 방화벽 차단, Proxy 설정 오류 방화벽 정책 확인, Proxy IP/Port/인증 정보 재입력
Device ID / Claim code 만료 5분 경과 후 자동 갱신 IQ Link Installer 페이지에서 새 코드 확인 후 재입력
DNS 조회 실패 DNS 서버 미등록 또는 A/PTR 레코드 누락 DNS IP 재확인, A 레코드 및 PTR 레코드 생성
NTP 동기화 실패 UDP 123 차단 또는 NTP 서버 미응답 NTP 서버 접근성 확인, 포트 123 허용
스토리지 용량 부족 경고 Thin Provisioning 사용으로 과할당 Thick Provisioning으로 재배포
Internal Network IP 충돌 입력한 IP 범위가 기존 장비와 겹침 사용 중인 IP 대역 확인 후 충돌 없는 범위로 재설정
ℹ️ 추가 지원이 필요한 경우 Cisco Support Case를 오픈하십시오. Cisco Service Contract가 필요합니다.
